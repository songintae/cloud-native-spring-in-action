# 레퍼런스 문서 템플릿

## 기본 레퍼런스 템플릿

```
# {주제명}

## 개요
핵심 정의 한두 문장 + 왜 필요한지.
🌱 Spring이 이 기능을 도입한 설계 동기를 설명한다.

## 핵심 동작 원리
ASCII art 시퀀스/흐름 다이어그램 + 단계별 설명.
🌱 Spring 내부 클래스 수준까지 추적한다.
   (예: 어떤 인터페이스가 어떤 순서로 호출되는지)

## 기본 사용법
Polar Bookshop 도메인 기반 코드 예시.
의존성 추가 → 설정 → 코드 순서로 작성.
코드 블록 내에 [실무노트] 주석을 삽입하여 현업 팁을 전달한다.

## 심화 내용
🌱 내부 구현 상세, 확장 포인트, 고급 설정.
   Spring이 제공하는 커스터마이징 인터페이스와 활용 예시.

## 실무 고려사항
🌱 Spring 관점: 흔한 실수, 성능 튜닝, 버전별 차이점.
☁️ Cloud Native 관점: 컨테이너 환경 주의점, 수평 확장 영향,
   12-Factor 원칙과의 관계, K8s 운영 시 고려사항.

## 정리
요약 테이블 (컴포넌트 | 역할) + 한줄 결론.
```

## 딥다이브 문서 템플릿

딥다이브 모드에서는 아래 Why-How-Pitfall 구조를 사용한다.
책(Cloud Native Spring in Action) 내용을 기반으로 하되, 실무에서 자주 만나는 관련 개념까지 확장하여 다룬다.

```
# {주제명} 딥다이브

## Why — 왜 필요한가
이 기술/패턴이 해결하는 근본 문제가 무엇인지.
🌱 Spring이 이 기능을 도입한 역사적 맥락과 설계 동기.
☁️ Cloud Native 환경에서 이 개념이 왜 더 중요해지는지.

## How — 내부에서 어떻게 동작하는가
### 동작 흐름
ASCII art 시퀀스/흐름 다이어그램으로 전체 흐름을 시각화한다.

### 핵심 내부 구현
🌱 Spring 내부 클래스/인터페이스 수준까지 추적한다.
   어떤 클래스가 어떤 순서로 호출되는지, 확장 포인트는 어디인지.

### 코드로 보기
Polar Bookshop 도메인 기반 코드 예시.
코드 블록 내에 [실무노트] 주석으로 "현업에서는 왜 이렇게 하는지" 이유를 담는다.

## Pitfall — 흔한 실수와 안티패턴
### 🚫 안티패턴 목록
| 안티패턴 | 왜 문제인가 | 올바른 접근 |
|---------|-----------|-----------|
| {패턴1} | {이유}     | {대안}     |

### 🌱 Spring 관점 주의사항
버전별 차이점, 설정 실수, 성능 함정 등.

### ☁️ Cloud Native 관점 주의사항
컨테이너 환경 특유의 문제, 수평 확장 시 깨지는 가정,
12-Factor 원칙 위반 사례, K8s 운영 시 주의점.

## 실무 확장
책에서 다루지 않지만 실무에서 자주 만나는 관련 주제.
(예: 책에서 Spring Data JDBC를 다루면 → 실무에서의 N+1 문제, 배치 처리 전략 등)

## 한줄 정리
핵심을 한 문장으로 요약.
```
