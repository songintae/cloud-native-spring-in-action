# rules/ 디렉토리

파일 패턴에 따라 조건부로 로드되는 규칙을 관리하는 디렉토리.
CLAUDE.md에 모든 규칙을 넣는 대신, 관련 파일 작업 시에만 해당 규칙이 자동 적용된다.

## 파일 구조

```yaml
---
paths:
  - "**/*.java"
  - "**/src/**/*.java"
---

# 규칙 제목

여기에 규칙 내용을 마크다운으로 작성한다.
```

## 동작 원리

- `paths`에 지정된 glob 패턴과 일치하는 파일을 작업할 때만 해당 규칙이 컨텍스트에 포함됨
- 여러 규칙 파일이 동시에 매칭될 수 있음 (예: `.java` 파일은 `java.md` 규칙 적용)
- `paths`를 생략하면 항상 로드됨 (= CLAUDE.md에 넣는 것과 동일)

## 현재 등록된 규칙

| 파일 | 매칭 패턴 | 역할 |
|------|-----------|------|
| `java.md` | `**/*.java` | 코드 컨벤션, 주석 패턴, 테스트 컨벤션 |
| `config.md` | `**/application*.yml`, `**/build.gradle` 등 | 설정 파일 주석 패턴, 외부화 원칙 |
| `kubernetes.md` | `**/deploy/**/*.yml`, `**/Dockerfile` 등 | 컨테이너/K8s 베스트 프랙티스 |

## 작성 가이드

- 파일명은 역할을 나타내는 이름으로 (예: `java.md`, `config.md`, `testing.md`)
- `paths` 패턴은 프로젝트 루트 기준 glob 형식
- 규칙 내용은 간결하게. Claude가 매번 읽으므로 불필요한 설명은 제외
- CLAUDE.md와 중복되지 않도록 주의 (CLAUDE.md = 프로젝트 전체 맥락, rules/ = 파일별 세부 규칙)

## 추가/수정/삭제

- 추가: 이 디렉토리에 `.md` 파일 생성 (frontmatter에 `paths` 지정)
- 수정: 해당 `.md` 파일 편집
- 삭제: 해당 `.md` 파일 제거 (다른 설정 변경 불필요)
